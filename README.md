# 2020.SRTP
2020.7-2021.5 -- The SRTP Qt project  
今天是2021.2.4，这是我第一个同步github的工程，当然也是验证自学成果的第一个工程  

该工程基于Qt开发，方向为高铁的动力投切  
涉及到Qt对excel的数据读入，PID控制等，是一项基于matlab的移植程序  

使用说明：  
1. 默认读取文档为Excel，记得填写文件后缀  
2. 文档数据的首行为空行，数据应从第2行开始  
3. .pro文件的eigen路径记得更改，不然会报错  
4. “数据”文件夹中有列车运行的相关数据，运行该程序时需要将表格拷背到QT编译后的文件夹中  

过程简记：  
3/10 移植过程中因为matlab中有太多的矩阵运算而遇到许多困难，尝试使用Eigen库移植matlab。  
3/25 采用Eigen库的移植程序已基本完成，不过Cal_resistence函数部分仍有问题，输出数据和matlab有较大差异，float型会因为填充未满的小数部分而随着迭代次数的增加放大误差，仍未想到有效解决办法，PID部分仍没debug，图形化界面仍没开始。  
3/31 Cal_resistence的数据问题已经解决，原因是表格数据初始化少了一步，现在开始调试PID，并开始写图形化界面子文件。  
5/5 赶在了五一最后一天完成了图显部分，基于qcustomplot第三方图形库开发，主要借鉴了example中plot部分的图显代码。还有几个图没做，excel的读表方案找到了一个解决办法，读表速度有了优化方案，目前而言2020的 SRTP 结题没有问题，剩下只有一些优化问题了。
